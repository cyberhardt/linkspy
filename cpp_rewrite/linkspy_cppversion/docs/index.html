Basic usage

The main window is split into 5 main parts:

The main toolbar

This is where you enter the URL to be checked, specify whether you want to use POST and select the useragent to use.

There are also a number of icons:

  • : this is the Go button
  • : Batch mode processing
  • : Shows program information
  • : Allows you to set program options

Once you've entered the URL you wish to check and pressed the Go button you'll be presented with a new tab. A tab is split into three more parts:

Secondary toolbar

This toolbar shows you the IP address and country flag (if geoip.db is available) of the current URL, it also contains buttons allowing you to perform a whois, ping or traceroute on the domain.

There are also two additional buttons:

  • : opens the current URL in your default web browser
  • : allows you to save the HTTP response to a file

Links found list

This is a list of all the URLs that have been found within the HTTP response, you can double click these to perform additional lookups.

HTTP header

This shows the HTTP headers returned from the server.

HTTP body

This shows the HTTP body returned from the server (usually the HTML content of a webpage).

HTTP status code and size of response

This is shown the main window status bar.

Making requests using the POST method

If you tick the Use POST checkbox linkspy will perform a POST request instead of a GET request. Before this request happens you will be presented with a form values editor.

You can also load and save POST variables and choose to clear them after use. Clicking OK on this window will perform the request.

The Useragent manager

The useragent manager is invoked by using the ... button next to the Useragent dropdown on the main toolbar and allows you to manage which useragents are available for use.

You can add, edit, delete or restore defaults which downloads the latest useragents from the linkspy website.

The system tray

Not only does linkspy minimise to the system tray, the icon will change status depending on whether any links have been found in the clipboard.

When there are no links waiting to be checked, the icon will be it's standard form: .

The icon becomes highlighted when links are available.

The tray icon also contains a context menu which contains the list of domains discovered through the clipboard as well as options to clear the list, restore the main window and exit the application.

Batch processing

Batch processing allows you to check several URLs in one go. There are two ways to activate the batch mode:

  • Pressing the batch button on the main toolbar and entering the URLs (one per line) into the dialog that appears
  • Copy the URLs to the clipboard and use the Check All option from the system tray
A new section appears in the main window and the URLs and their response codes & sizes will appear as they are completed. This section has its own Save button allowing you to save the results as a CSV.

Domain & network tools

linkspy allows you to perform various operations on links:

  • ip: gives you the IP address of the remote host
  • geoip: if the geoip.db file is present in the exe directory the application will show the country flag of the host
  • whois: return whois information about the current domain
  • ping: pings the current host 5 times and returns the response times
  • traceroute: performs a traceroute to the current host (maximum of 15 hops)

Program options

This window allows you to setup the font and colours of the HTTP response aswell as clear your URL history.
